# Query Planning

Start with the user's words, then add likely project or decision terms. Good recall plans include:

- Narrow scope: project path, source, session, or time window if known.
- Query variants: exact phrase, domain synonym, artifact/file name, and decision verb.
- Drill-down decision: why a hit deserves `get`, `context`, or `session --around`.
- Stop condition: enough cited evidence, or clear gap after reasonable broadening.

Avoid:

- Searching only one vague term then declaring no memory.
- Paging a whole session from the top.
- Treating high search rank as truth without context.
- Omitting failed queries; failed retrieval is useful evidence.
